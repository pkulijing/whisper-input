# 产品正规化：UX 反馈 + 产品信息

## 背景

Whisper Input 功能已基本完善，但在用户体验和产品信息展示方面还不够正规：

1. **缺乏状态反馈**：启动时模型加载需要 1-2 秒，期间无任何视觉提示，用户以为软件卡了；按住快捷键录音时也没有提示，用户不确定快捷键是否按对、软件是否在工作。微信输入法在这方面做得很好——按快捷键桌面上会浮现话筒图标。
2. **产品信息缺失**：托盘菜单只有"设置"和"退出"，缺少版本号等基本产品信息；设置页面也没有 GitHub 链接等开发者沟通渠道。
3. **打包分发体验差**（本轮暂不处理）：软件本体仅 500 多 K，但需要安装依赖库、下载模型，体验不佳。

## 需求

### UX 反馈

1. **托盘图标状态变化**：
   - 灰色 — 启动中/模型加载中
   - 绿色 — 就绪
   - 红色 — 录音中
   - 橙色 — 识别中

2. **录音浮窗**：
   - 按住热键时，屏幕上弹出半透明浮窗，显示话筒图标和状态文字
   - 松开热键后显示"识别中..."，识别完成后浮窗消失
   - 类似微信语音输入的体验

3. **设置开关**：
   - 浮窗和托盘图标状态变化都可以在设置页面中开关
   - 默认都开启

### 产品信息

1. **托盘菜单版本号**：菜单顶部显示 `Whisper Input v{版本号}`，不可点击
2. **设置页面版本号**：页面底部展示版本号
3. **GitHub 链接**：设置页面底部展示 GitHub 项目链接

## 本轮不做

- 打包分发优化（PyInstaller 打包、模型懒加载等），留给下一轮
